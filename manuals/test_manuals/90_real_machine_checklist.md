# 90 Real Machine Checklist

## 目的
- 自動テスト通過後に、実機で見落としやすい統合不具合を捕捉する

## 事前条件
- 最新コードでサーバー起動済み
- ログ表示UIと詳細ポップアップを確認できる状態

## チェック項目
1. フェーズ遷移
- mass -> single -> round_end が毎ラウンドで成立する
- ラウンド終了後に入力UIが再初期化される

2. ログ整合
- 結果ログ、詳細ログ、traceポップアップで情報が矛盾しない
- 同一イベントが重複登録されない

3. ダメージ/状態
- HP/MP/FP の遷移がログとUI表示で一致する
- 付与/解除、継続効果、再使用の順序が意図通り

4. 分岐系
- clash 引き分け
- one-sided 不発
- hard followup 発火/不発
- self destruct 発火

5. 長時間安定性
- 5ラウンド以上連続で処理し、進行停止やログ破綻がない
- PvE 自動進行で意図宣言が詰まらない

## NGシグナル
- phase が `resolve_single` で固定される
- ログは進むが HP/状態が更新されない
- 1イベントで同一ログが複数回 append される
- ラウンド終了後に前ラウンド intent が残留する
