# 10 凝魔（特殊バフ型リソース）実装計画

**作成日**: 2026-03-29  
**最終更新日**: 2026-04-04  
**対象**: 新規特殊バフ `凝魔`  
**位置づけ**: 実装前の仕様確定用計画書

---

## 1. 目的

`凝魔` を、特定スキルで獲得し、残り回数を消費して追加効果へ変換する **特殊バフ** として実装する。  
本計画では、名称統一・分類・回数管理・表示・効果処理を一貫して定義する。

---

## 2. 基本方針

- 名称は一律で `凝魔` を使用する（旧名称 `蓄魔` は廃止）
- `凝魔` は状態異常ではなく **特殊バフ** として扱う
- スタック数は「残り回数（count）」として扱う
- 継続ラウンドは原則無制限（`lasting` が未設定の場合）
- `lasting` を設定した場合のみ、ラウンド減衰を適用する
- ラウンド減衰の実装方針は、既存の亀裂ラウンド管理ロジックの扱いに準拠する

---

## 3. 分類とデータモデル

## 3.1 凝魔の分類

`凝魔` は次の分類で固定する。

- `buff_class`: `special_resource`
- `display_group`: `buff`
- `is_negative`: `false`

表示説明文（固定）:

`特定のスキル効果によって獲得する。残り回数を消費し、一部スキルの追加効果を適用することができる。`

## 3.2 凝魔インスタンスの管理項目

- `count`: 残り回数（消費成功時に減る）
- `lasting`: 継続ラウンド（未設定なら無制限）
- `delay`: 必要時のみ使用（通常は0）

ルール:

- `count` が 0 以下になった時点で `凝魔` を除去
- `lasting` が未設定 (`null` / 欠損) の場合、ラウンド経過で消えない
- `lasting` が設定されている場合のみ、ラウンド終了で減算し 0 で除去

## 3.3 既存5状態異常の分類固定（回帰防止）

既存5状態異常（`出血`・`破裂`・`亀裂`・`戦慄`・`荊棘`）は、従来どおり次で固定する。

| 状態異常 | status_class | display_group | is_negative |
| --- | --- | --- | --- |
| 出血 | `debuff` | `debuff` | `true` |
| 破裂 | `debuff` | `debuff` | `true` |
| 亀裂 | `debuff` | `debuff` | `true` |
| 戦慄 | `debuff` | `debuff` | `true` |
| 荊棘 | `debuff` | `debuff` | `true` |

補足:

- `凝魔` は上表の5状態異常に含めない
- デバフ解除処理は上表のみを対象にし、`凝魔` は対象外とする

---

## 4. 効果定義（獲得・消費・獲得量）

## 4.1 凝魔を獲得する効果

推奨:

- `APPLY_BUFF` で `buff_name: "凝魔"` を付与
- 既に `凝魔` がある場合は `count` を加算

## 4.2 凝魔を消費して効果を得る定義

消費要求値と獲得量は分離して数値定義する。

- `CONSUME_BUFF_COUNT_FOR_GAIN`
- `buff_name`: `凝魔`
- `consume_required`: 必要回数
- `gains`: 成功時獲得（FP/MP/バフなど）

判定:

- `current_count >= consume_required` のときのみ成功
- `consume_required > current_count` の場合、**消費も獲得も発生しない（不発）**

例1（凝魔3消費でFP+1）:

```json
{
  "timing": "HIT",
  "type": "CONSUME_BUFF_COUNT_FOR_GAIN",
  "target": "self",
  "buff_name": "凝魔",
  "consume_required": 3,
  "gains": [
    { "type": "FP", "value": 1 }
  ]
}
```

例2（凝魔4消費でFP+2）:

```json
{
  "timing": "HIT",
  "type": "CONSUME_BUFF_COUNT_FOR_GAIN",
  "target": "self",
  "buff_name": "凝魔",
  "consume_required": 4,
  "gains": [
    { "type": "FP", "value": 2 }
  ]
}
```

---

## 5. 表示とログ

- `凝魔` はバフアイコンとしてキャラ駒右上に表示
- 表示値は `凝魔 x<count>`
- ツールチップ説明文は固定文言を表示
- ログは最低限「獲得」「消費成功」「不足不発」を区別して出力

---

## 6. 実装フェーズ

## Phase A: 仕様反映

- 名称を `凝魔` に統一
- `凝魔` の分類を特殊バフとして定義
- 固定説明文をデータ定義へ反映

## Phase B: ロジック実装

- `APPLY_BUFF` 時の `凝魔 count` 加算処理
- `CONSUME_BUFF_COUNT_FOR_GAIN` の追加
- 不足時は不発（消費なし・獲得なし）を保証

## Phase C: ラウンド管理

- `lasting` 未設定時は無制限継続
- `lasting` 設定時のみ減衰
- 亀裂ラウンド管理ロジックと同じ減衰タイミング・除去条件で統一

## Phase D: UI/表示

- 駒右上のバフアイコン表示に `凝魔` を追加
- 固定説明文の表示確認

## Phase E: データ運用

- Buff Catalog / Glossary / スキル正本の名称・説明を `凝魔` に更新
- 既存データの `蓄魔` 記述を置換

---

## 7. テスト観点

1. `凝魔` 付与時にバフとして登録される
2. 既存 `凝魔` がある状態で付与すると `count` が加算される
3. `consume_required=3` で `count=3` のとき成功し、FP+1 を得る
4. `consume_required=4` で `count=4` のとき成功し、FP+2 を得る
5. `consume_required > count` のとき不発（消費なし・獲得なし）
6. `lasting` 未設定時はラウンド経過で消えない
7. `lasting` 設定時はラウンド経過で減衰し 0 で除去される
8. 既存5状態異常が `debuff / debuff / true` を維持する
9. UIで `凝魔 x<count>` と固定説明文が表示される

---

## 8. 受け入れ条件

1. 旧名称 `蓄魔` が計画・データ定義から除去され、`凝魔` に統一されている
2. `凝魔` が特殊バフとして実装方針化されている
3. 回数管理（`count`）と継続管理（`lasting`）の仕様が明文化されている
4. `lasting` 未設定時の無制限継続が仕様・テストで保証されている
5. 消費不足時の不発ルール（消費なし・獲得なし）が定義されている
6. 固定説明文が表示仕様として明記されている
