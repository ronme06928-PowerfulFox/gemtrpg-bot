# 16. 増援機能 別計画
作成日: 2026-05-07

## 位置づけ
増援機能は、今回の通常プリセット/戦闘専用ステージ導線の実装範囲から外します。
本書は、後続実装のために仕様を分離して整理する計画書です。

## 対象プリセット
増援設定を持てる候補は以下です。

- 敵編成プリセット
- ステージプリセット

敵キャラ単体プリセットには、原則として増援投入条件を持たせません。
敵キャラ単体は、増援として投入される素材として扱います。

## 増援定義の基本形
増援定義は、投入条件と投入内容を分離します。

```json
{
  "reinforcements": [
    {
      "id": "reinforcement_1",
      "timing": "round_start",
      "condition": {
        "type": "round_gte",
        "value": 3
      },
      "members": [
        {
          "preset_id": "enemy_1",
          "count": 2
        }
      ],
      "once": true
    }
  ]
}
```

## 条件候補
初期候補は以下です。

- ラウンド条件: `round_gte`, `round_eq`
- 敵キャラHP割合条件: `enemy_hp_ratio_lte`
- 敵全体HP割合条件: `enemy_total_hp_ratio_lte`
- 特定敵撃破条件: `enemy_defeated`

HP割合は `0.0` から `1.0` の小数で扱う方針を第一候補にします。

## 投入タイミング候補
初期候補は以下です。

- ラウンド開始時
- ラウンド終了時
- 条件成立直後

最初の実装では、処理順が安定しやすい `ラウンド開始時` を優先します。

## 投入内容
投入内容は敵編成プリセットの `members` と近い形式にします。

- `preset_id`
- `count`
- 任意の行動チャート上書き
- 任意の初期位置指定

初期位置未指定の場合は、既存の敵配置ロジックを使います。

## 実装しないこと
この計画段階では、以下は実装しません。

- UI実装
- 戦闘中の自動投入処理
- 保存済みデータのマイグレーション
- E2Eテスト
