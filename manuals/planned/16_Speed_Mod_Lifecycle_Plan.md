# 16. Speed Mod Lifecycle Plan

**最終更新日**: 2026-04-19  
**対象バージョン**: Current  
**対象機能**: 加速/減速の消費ロジック改善（当ラウンド適用分と次ラウンド予約分の分離）

---

## 1. 目的
- 加速/減速を「速度値へ適用済みの同ラウンド内バフ」と「次ラウンドで効く予約分」に分けて管理する。
- `lasting` / `delay` の標準運用で将来の速度条件スキル実装を安定化する。

---

## 2. 現状
### 2.1 実装済み
- `SpeedModBuff.get_speed_modifier` で `Bu-11/Bu-12` を合算
- ラウンド開始時に速度へ反映

### 2.2 課題
- `SpeedModBuff.clear_speed_modifiers` により速度反映直後に全削除される
- 当ラウンド適用分と次ラウンド予約分を同時保持できない

---

## 3. 実装方針
### 3.1 バフライフサイクル
- 当ラウンド適用分: `delay=0`, `lasting=1`
- 次ラウンド予約分: `delay=1`, `lasting=1`
- 速度計算では `delay==0` のみ参照
- ラウンド終了処理で
  - 適用分は `lasting 1->0` で消滅
  - 予約分は `delay 1->0` で次ラウンド有効化

### 3.2 スタック管理
- `buff_id` だけでなく `delay` 単位でバケット分離
- これにより「今効いている値」と「次に効く値」を別参照可能

### 3.3 既存ロジック変更点
- `clear_speed_modifiers` 呼び出しを廃止または no-op 化
- `get_speed_modifier` を `delay` / `lasting` 考慮に拡張

---

## 4. 実装タスク
1. `plugins/buffs/speed_mod.py` の apply/get/clear 改修
2. ラウンド開始処理のクリア呼び出し整理
3. ラウンド終了処理の挙動確認（既存 delay/lasing 更新と整合）
4. 表示用ヘルパ（必要なら「適用中」「予約中」）追加

---

## 5. テスト観点
- Rラウンドで適用された加減速が速度に反映される
- 同ラウンド中は「適用中」として参照できる
- `delay=1` の予約分が R+1 で反映される
- 予約分と適用中分が混ざらず別カウントで管理される
