# Battle Balance Designer Skill 運用マニュアル

最終更新: 2026-07-07

## 目的

`battle-balance-designer` は、TRPG/ダイス戦闘のバランスと面白さを相談するための Codex Skill である。

単に「強い/弱い」を判定するのではなく、次の観点を一緒に見る。

- プレイヤーの選択に意味があるか
- 勝敗やダメージ結果に納得感があるか
- 戦闘テンポが重すぎないか
- 防御、支援、召喚、妨害、火力の役割が成立しているか
- GMやプレイヤーの処理負荷が卓で扱える範囲か
- Gem_DiceBotTool の Select/Resolve、clash、one-sided、召喚、FP、バフスタック、PvE自動意図と矛盾しないか

明示的に使いたい場合は、依頼文に `$battle-balance-designer` を含める。

例:

```text

狙いは「支援役が一瞬だけ火力役を輝かせる」です。
```


## 暗黙的な読み込みについて

Codex の Skill は、常に全文が読み込まれているわけではない。

通常は、各 Skill の `name` と `description` がトリガー判定に使われる。依頼内容が description に合うと判断された場合、Codex がその Skill の `SKILL.md` を読み込んで使う。

この Skill は description に次のようなトリガー語を入れている。

- TRPG
- ダイス戦闘
- PvE遭遇
- スキルカタログ
- バフ/デバフ
- 召喚
- 行動経済
- clash/duel
- Gem_DiceBotTool
- 戦闘バランス
- ゲームの面白さ
- 戦闘テンポ

そのため、「このスキル強すぎる？」「PvE敵の行動を調整したい」「召喚が壊れないようにしたい」のような相談では、明示しなくても呼ばれる可能性が高い。

ただし、確実に使わせたい場合は `$battle-balance-designer` と書くのが安全である。

## 相談時にあるとよい情報

相談時は、全部を揃える必要はないが、次があるとレビュー精度が上がる。

- 作りたい面白さ
- 想定プレイヤー人数
- 想定ラウンド数
- 既存スキルとの比較対象
- コスト、対象、タイミング、継続時間、使用制限
- 失敗してほしくない体験
- その案を使う役割: 火力、防御、支援、妨害、召喚、PvE敵など

## 返答の基本形

大きめのレビューでは、Skill は次の形で返すようにしている。

```text
狙い:
この案が作りたい体験。

強い点:
面白さや役割として成立している部分。

リスク:
支配的行動、雪だるま化、処理負荷、リソース黒字、行動経済破壊など。

調整案:
小さく試せる変更案。各案のトレードオフも書く。

検証:
プレイテストや簡易計算で見るべきこと。
```

短い相談では、見出しを省略して会話形式で答える。

## Gem_DiceBotTool で特に見る観点

Gem_DiceBotTool の戦闘では、次を重く見る。

- `clash` はダメージだけでなくFP獲得や防御/回避の価値に関わる。
- `one_sided` は対抗不能時の圧力を作るため、対象選択とテンポに影響する。
- 召喚は追加ユニットだけでなく、将来の行動スロットと対象数を増やす。
- `USE_SKILL_AGAIN` は追加行動に近く、行動価値を大きく増幅する。
- バフ/スタック資源は、上限、持続、掃除、表示を同時に設計する必要がある。
- PvE自動意図と敵AIは、プレイヤースキルを変えなくても遭遇難度を大きく変える。
- resolve log で説明できない効果は、プレイヤーの納得感を落としやすい。

この Skill は、最終的な正解を固定するためのものではなく、設計判断の質を上げるための文脈である。

数値だけで結論を出さず、次のように扱う。

- まず「何を面白くしたいか」を確認する。
- 次に「どの条件で壊れるか」を見る。

特に、追加行動、召喚、無料リアクション、無制限スタック、リソース黒字ループは、初期案では控えめに設計する。

## スキルバランス調整基準（確定版・2026-06-28）

*Plan 24 / Plan 25 より統合。スプレッドシート反映済み。今後は実プレイで問題が出た箇所を個別調整する。*

### 基本前提

- 標準戦闘: 8ラウンド程度。味方HP約120。能力補正は+4で統一して除外。
- 基本技は最低限の攻防手段（単なるリソース回復行動ではない）。

| 敵区分 | 想定 |
|---|---|
| 雑魚 | 複数体・戦闘全体4ラウンド程度 |
| 中堅 | 1〜2体・8ラウンド程度 |
| 強敵・ボス | 10ラウンド以上。段階変化や行動パターンで長さを作る |

敵火力: 無抵抗時に5ラウンドで危険域（累計被害100前後）が上限目安。
回復: 控えめ。同コストの攻防より安全な選択にならないよう量・条件・コストで抑える。
守備系: マッチ勝利でもダメージを与えないため、同威力・同コストの攻撃より軽く評価する。

### 威力段階

| 段階 | 役割 | 中心威力 |
|---|---|---:|
| 1 | スターター・軽い主力 | 8 |
| 2 | 繋ぎ・お膳立て・主力 | 11 |
| 3 | 大技・決定打 | 15 |

### 使用コストの基準

基本技の機会費用を含めて評価する（物理基本技FP+2、魔法基本技FP+1 MP+1）。

| 区分 | 目安 |
|---|---|
| 物理 段階1 | FP1前後 |
| 物理 段階2 | FP3前後 |
| 物理 段階3 | FP5前後 |
| 魔法 段階1 | MP2 / FP0前後 |
| 魔法 段階2 | MP3 / FP1〜2前後 |
| 魔法 段階3 | MP6 / FP3前後 |

上下する要素: 的中時か使用時か / 即時か遅延か / 単体完結か後続依存か / 状態異常の即効性 / 副作用・対象制限。

### 状態異常の評価

- **出血**: 標準状態異常。出血4〜5的中時付与は取得1スターターの標準範囲。
- **破裂**: 起爆が必要なため、付与数値はやや大きめに取る。
- **亀裂**: 攻撃回数増で膨れやすい。継続ラウンド・ラウンド付与制限・低付与数値で抑える。
- **減速・挑発・被ダメ増**: 高価値。副作用・遅延があれば相殺。
- **蓄力・凝魔**: 後続消費で初めて価値が出るため即時より軽く見る。
- **再使用・変換・遷延**: ビルドの動きを変えるため高価値。条件・準備・追加消費で相殺。

### 取得コストの基準

取得コストはスキル段階と機械的に連動させない。
新規作成時の平均 8Pt 程度、最低寄り 5Pt、最大 10Pt 程度を想定。

| 取得 | 意味 |
|---:|---|
| 0 | 基本技・共通前提 |
| 1 | ビルドの基本パーツ |
| 2 | 主力・中核・強い展開札 |
| 3 | 希少な個性化枠（ビルドの質を変える） |
| 4以上 | 例外。特殊取得・上位解放・成長後向け |

**取得3の基準**: 既存ビルドの質を変える / コンボを大きく拡張する / 強い持続・変換・踏み倒し・放出・遷延 / サービス序盤の通常大技ではなく後の成長段階の技。

### 横断的な判断基準

- **自傷・自己リスク**: 単なる軽いコストとして扱わない。HP消費・継続ダメージ・将来リスクを含めて評価。
- **複合キーワード**: 準備スキル取得・パーティー協力・蓄積コストを含めて評価。条件が重ければ取得1〜2許容。
- **セット運用**: 単体でなくセット全体の取得コストで見る。
- **属性タグ差別化**: 斬撃・打撃・貫通で数値差はあってよい。将来パッシブで各属性を選ぶ理由を作る。

### 調整時の運用ルール

1. まず使用コストを調整する。
2. 次に状態異常量・継続ラウンド・成立条件を調整する。
3. それでも役割が過剰なら取得コストを調整する。
4. 取得3以上は通常の強スキルではなく、ビルドの質を変える希少枠に限定する。

現時点では取得コストの大規模な再配分は行わない。

## スキル効果・威力設計ガイド（GM・コンテンツ作成者向け）

*2026-07-07 追加。数値基準の正本は上記「スキルバランス調整基準（確定版・2026-06-28）」。本章はその基準を新スキル作成の実作業に落とし込む実践ガイドであり、相場表は `data/cache/skills_cache.json`（2026-07-07 時点・全133件）の集計に基づく。マスターデータ更新後は相場が変わりうる点に注意。*

### 設計手順（5ステップ）

1. **狙いを1文にする**: 「支援役が一瞬だけ火力役を輝かせる」のように、作りたい体験を先に言語化する。数値から始めない。
2. **役割と系統を決める**: 火力/繋ぎ/お膳立て/守備/妨害のどれか。系統（Ps/Pb/Pp/Ms/Mb/Mp）と属性（斬撃/打撃/貫通）、距離（近接/遠隔/広域）を決める。
3. **威力段階を選び相場に合わせる**: 確定基準の段階1=8 / 段階2=11 / 段階3=15 を中心威力とし、下の相場表から外れる場合は理由（成立条件の重さ・副作用）を持つ。
4. **使用コストと取得コストを決める**: 確定基準のコスト表を起点に、的中時か使用時か・即時か遅延か・単体完結か後続依存かで上下させる。
5. **壊れ方を先に確認する**: 後述のチェックリストで「どの条件で支配的になるか」を具体化してから確定する。

### 現行カタログの構成（2026-07-07 時点）

| 系統 | 件数 | 内容 |
|---|---:|---|
| Ps / Pb / Pp | 15 / 16 / 15 | 物理（斬撃/打撃/貫通）。コストは FP 主体 |
| Ms / Mb / Mp | 15 / 14 / 15 | 魔法（斬撃/打撃/貫通）。コストは MP+FP 併用 |
| E | 31 | 敵専用。広域・自爆・強硬・牽制・召喚・伝授などギミック持ち |
| D | 7 | 守備（防御4・回避3） |
| B | 4 | 宝石の加護（即時発動の補助） |
| C | 1 | 魔力回復（マッチ不可・自己対象） |

取得コスト分布は 0=44 / 1=59 / 2=29 / 3=1。**取得1が主力ゾーン、取得3は Ms-07 の1件のみ**であり、「取得3以上はビルドの質を変える希少枠」という確定基準が実データでも守られている。新スキルもこの分布を崩さないこと（取得2を乱発しない）。

### 期待威力の相場（基礎威力＋ダイス期待値、PC用 P/M 系のみ）

| 区分 | 件数 | 期待威力平均 | 上限付近 |
|---|---:|---:|---:|
| 物理 取得1 | 28 | 約8.3 | 12.0 |
| 物理 取得2 | 15 | 約10.8 | 14.0 |
| 魔法 取得1 | 28 | 約8.8 | 22.0（Ms-09 瀉血=HP6消費の例外） |
| 魔法 取得2 | 12 | 約12.2 | 16.0 |

確定基準の威力段階（8/11/15）と実データはほぼ一致している。新スキルの期待威力（基礎+ダイス面数÷2+0.5）がこの帯を超える場合は、HP消費・後続依存・成立条件などの明確な代償を付けること。

使用コストの実勢: FP2（31件）と FP1（24件）が物理の中心、MP2（17件）と MP3（11件）が魔法の中心。HPコストは3件（5/6/8）のみで希少。FP4/MP6 は段階3級の重コスト帯。

### 効果タイミングの相場と使い分け

| timing | 件数 | 設計上の意味 |
|---|---:|---|
| HIT（的中時） | 101 | 標準。数値は確定基準の相場どおり |
| PRE_MATCH（使用時） | 33 | 的中不要で確実に成立するため、同じ数値でも HIT より強い。自己バフ・準備向け |
| WIN（マッチ勝利時） | 18 | 成立条件が最も重いため、HIT より1〜2段強い数値を許容できる（例: Pb-03 破裂10） |
| END_ROUND | 12 | 基本技のリソース回復など、遅延リターン |
| LOSE / UNOPPOSED / END_MATCH | 少数 | 敗北補償・非対抗時ボーナスなどの補助設計 |

### 状態異常の付与量相場（APPLY_STATE 集計）

| 状態異常 | 件数 | 平均 | 実勢レンジ | 設計指針（確定基準と対応） |
|---|---:|---:|---|---|
| 出血 | 23 | 4.8 | HIT で 3〜6 が中心、WIN で 2（低め）、敵用は 8〜10 まで | 標準状態異常。取得1スターターは出血4〜5的中時が基準 |
| 破裂 | 18 | 5.0 | HIT で 2〜8、WIN で 6〜10 | 起爆が必要なぶん出血よりやや大きい数値を許容 |
| 亀裂 | 16 | 1.4 | **1〜3 で厳格に管理** | 攻撃回数で膨れるため、低数値・継続制限で抑える方針が実データにも表れている |
| 戦慄 | 3 | 4.7 | 3〜6 | 事例が少ない。追加時は既存3件と比較する |
| 荊棘 | 1 | 8.0 | 敵専用（E-12） | PC 用に持ち込む場合は要バランス相談 |

### 効果タイプ別の注意（実装済み構造の使用実績）

- `APPLY_STATE`（81件）と `APPLY_BUFF`（42件）が主流。まずこの2つで表現できないか考える。
- 蓄積消費系（`CONSUME_BUFF_COUNT_FOR_GAIN` 7件、`CONVERT_STACK_RESOURCE_VARIANT` 2件など）は「溜める技＋放つ技」のセット運用で評価する（確定基準「セット運用」参照）。
- `USE_SKILL_AGAIN` は現在3件のみ（Ps-04 乱斬 / Mp-09 五月雨式 / Mp-14 よく味わえ）。**追加行動に等しく行動経済を直接増幅するため、新規追加時は必ずバランス相談を通す。**
- `SUMMON_CHARACTER` / `GRANT_SKILL` は敵専用が各1件（E-19 / E-21）。PC 側解禁は行動スロット増加の影響が大きく、単独判断で追加しない。
- E-15（必殺デバッグ撃・威力9999）はデバッグ専用。E-06（威力99の防御）は敵ギミック用の極端値。**PC 用スキルの参考値にしないこと。**

### 壊れやすい構造チェックリスト（確定前に必ず見る）

## 戦闘遭遇・シナリオ設計ガイド（GM向け）

*2026-07-07 追加。戦闘遭遇（敵編成・行動チャート・ステージ効果）の作成指針。探索パートや物語構成は対象外。機能の操作手順は `A02_GM_Creator_Manual.md`、モード仕様は `D01_Play_Mode_Specs.md` を参照。*

### 遭遇設計の逆算手順

確定基準の前提（標準戦闘8ラウンド・味方HP約120・敵火力は無抵抗5Rで累計100前後が上限）から逆算する。

1. **遭遇の役割を決める**: この戦闘でプレイヤーに何を体験させたいか（消耗・ギミック攻略・ボス撃破・時間制限など）。
2. **想定ラウンド数を決める**: 雑魚=戦闘全体4R程度 / 中堅=8R程度 / ボス=10R以上（段階変化で長さを作る）。
3. **敵HPを決める**: PC1人の1ラウンド期待与ダメは主力スキル（取得1、期待威力8前後）＋能力補正が目安。パーティ人数×想定ラウンド数×1人あたり期待与ダメから敵側総HPの初期値を出し、マッチでの相殺・防御行動を見込んで2〜3割減らす。
4. **敵火力を決める**: 「無抵抗のPCが5Rで危険域（累計100前後）」を敵全体の合計で超えないよう配分する。敵が多いほど1体あたりは軽くする。
5. **プレイテストで調整する**: 低ロール・平均・高ロールの3通りで撃破ターンを概算し、想定ラウンドから大きくズレたら HP→火力→ギミックの順で調整する。

### 敵スキル（E系統）の使い分け

E系統31件は敵の「役割」を作る部品として使う。実例パターン:

| パターン | 実例 | 効果 |
|---|---|---|
| 素の攻撃 | E-00〜E-05（威力帯 2〜6+ダイス） | 雑魚の基本行動。状態異常も軽め（亀裂1・出血3程度） |
| 広域圧力 | E-11 / E-13 / E-14 / E-30 | 全体にプレッシャーを与えるボス技。E-14 は出血10と重い |
| 準備→大技 | E-16（ちょっとした準備） | 予告と組み合わせて「次のターンに備える」読み合いを作る |
| ギミック | E-22（自爆）/ E-23（強硬）/ E-24（牽制）/ E-06（極端防御） | 遭遇の個性付け。乱用すると処理負荷が上がる |
| 行動経済系 | E-19（召喚）/ E-21（伝授） | 敵の行動スロットを増やす。撃破優先度の判断を生むが、長期化リスクに注意 |

### 行動チャート（behavior_profile）の設計パターン

仕様は `D01_Play_Mode_Specs.md` を参照。設計の型:

- **固定ループ型**: 「通常攻撃×2→大技」のような周期。プレイヤーが周期を読めるため、雑魚〜中堅に向く。
- **条件遷移型**: HP閾値や経過ラウンドでループを切り替える。ボスの「第2形態」はこれで作る。段階ごとに脅威の種類（火力→広域→ギミック）を変えると単調化しない。
- **予告の読み合い**: PvEモードの行動予告矢印は「次に誰が狙われるか」の情報公開。予告を見て守備・妨害スキルを使う判断が生まれるよう、痛い攻撃ほど予告が意味を持つ配置にする。

### ステージ効果の使いどころ

`field_effect_profile.rules` の3タイプ（仕様は `D01` Part3）:

| ルール | 使いどころ |
|---|---|
| `SPEED_ROLL_MOD` | 行動順の揺らぎを作る。テンポの単調化対策 |
| `DAMAGE_DEALT_MOD` | 属性テーマの演出（例: 特定属性が有利な地形）。ビルド選択に意味を持たせる |
| `APPLY_STATE_ON_CONDITION` | ラウンド経過やHP条件で状態異常を付与する環境ギミック。時間制限の演出に使える |

ステージ効果は「全員に平等にかかる圧力」なので、PC側だけが一方的に不利になる設定は納得感を落とす。対策手段（属性の選択・立ち回り）が残る強度に留める。

### 緊張感カーブの作り方

- **序盤（1〜2R）**: 様子見と情報公開。敵の行動パターンを見せる。ここで大技を撃たない。
- **中盤（3〜6R）**: リソース判断の山場。FP/MPの使いどころ、状態異常の蓄積、守備と攻撃の択を作る。
- **終盤（想定ラウンド±2R）**: 決定打の応酬。負けている側に読み取れる逆転ルート（起爆・蓄積解放・温存した大技）が残っているかを確認する。
- 増援による起伏づけは `planned/16_Reinforcement_Feature_Separate_Plan.md`（未実装）の管轄。現状は GM の NPC 手動追加で代替する。

### GM負荷・テンポの上限

- 敵数×状態異常の管理量が卓の処理能力を超えないようにする。状態異常を持つ敵は編成の半分以下が目安。
- 1体に複数ギミック（強硬＋広域＋召喚など）を積むより、1体1ギミックで役割分担するほうが読みやすく処理も軽い。
- resolve ログで説明できない効果・例外処理をシナリオ側で作らない（自作の口頭ルールは納得感と再現性を落とす）。

### 遭遇チェックリスト

- [ ] 想定ラウンド数と敵HP・火力が逆算式から大きく外れていないか
- [ ] 低ロール時に「詰み」（勝ち筋消失）が発生しないか
- [ ] 高ロール時に想定の半分以下のラウンドで終わらないか（ボスのみ重視）
- [ ] 毎ラウンド同じ行動が最適にならないか（択が残るか）
- [ ] 負けている側に逆転ルートが読み取れるか
- [ ] 守備・支援・妨害の役割に見せ場があるか
- [ ] デバッグ用スキル（E-15等）が編成に混入していないか
