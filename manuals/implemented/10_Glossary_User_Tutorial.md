# 用語図鑑（Glossary）機能 チュートリアル

**最終更新日**: 2026-02-21  
**対象**: プレイヤー / GM / データ入力担当

---

## 0. このチュートリアルの目的

このドキュメントは、用語図鑑機能を使う人向けに、次の2点を最短で把握するための手引きです。

1. プレイ中に用語をどう参照するか  
2. スキル説明文へ用語リンク/スキルリンクをどう埋め込むか

---

## 1. プレイヤー向け: 用語の見方

用語リンクが有効な画面では、スキル説明内の対象語句をクリック（またはタップ）すると説明が表示されます。

- PC: hoverで短文、clickで詳細
- Mobile: tapで詳細
- 閉じる: 画面外クリックまたは `Esc`

表示の優先順:

1. tooltip（hover想定）: `short`（短文説明）優先
2. popup（click/tap想定）: `long`（本説明）優先、未登録時は `short`
3. 両方未登録時は「説明未登録」

---

## 2. データ入力者向け: 埋め込み記法

スキル説明文に以下の記法を書きます。

### 2.1 用語リンク記法

- `[[TERM_ID|表示名]]`
- `[[TERM_ID]]`（表示名省略）

例:

- `対象に[[W-00|出血]]を2付与する。`
- `自身が[[W-12]]状態なら威力+2。`

### 2.2 スキル詳細リンク記法

- `[[SKILL:SKILL_ID|表示名]]`
- `[[SKILL:SKILL_ID]]`（表示名省略）

例:

- `この効果で[[SKILL:E-20|見よう見まねで]]を付与する。`
- `次に使う[[SKILL:Mp-06]]の威力を強化する。`

注意:

- `SKILL:` プレフィックスは必須（`[[E-20]]` はスキルリンクになりません）
- 無効なIDでもクリック時はID表示でスキル詳細モーダルを開きます
- キャラ文脈がある画面では、キャラ側で改名したスキル名を優先して表示します（未改名時はデフォルト名称）

埋め込み先の推奨列:

- `使用時効果` / `コスト`
- `発動時効果` / `効果`
- `特記`
- `チャットパレット`

---

## 3. スプレッドシート入力ルール（用語図鑑シート）

`関連用語ID` と `別名` はカンマ区切りで入力します。

- `関連用語ID`: `W-00,W-01,W-15`
- `別名`: `Bleed,継続ダメージ`

補足:

- 区切り文字は `,` と `、` のどちらでも可
- `関連効果ID` ではなく `関連用語ID` を使用

## 4. よくある質問

### Q1. `[[W-00]]` と `[[W-00|出血]]` の違いは？

- `[[W-00]]`: 図鑑の `display_name` を表示  
- `[[W-00|出血]]`: 表示文字を手動指定

### Q1-2. `[[SKILL:E-20]]` と `[[SKILL:E-20|見よう見まねで]]` の違いは？

- どちらもクリックでスキル詳細モーダルを開く  
- `|表示名` ありは本文の表示文字を手動指定  
- `|表示名` なしはデフォルト名称を使用（キャラ文脈がある画面では改名名を優先）

### Q2. 用語リンクが表示されない

次を確認してください。

1. 用語IDが図鑑シートに存在する
2. `表示の有無` が `TRUE`
3. 図鑑データ更新後にアプリを再読込している
4. 該当画面がGlossary UI統合済みである

### Q3. ツールチップが「説明未登録」になる

次を確認してください。

1. 図鑑シートの `短文説明` または `本説明` が空欄でない
2. 用語IDが一致している（例: `W-00` と `w-00` は別扱い）
3. シート更新後にハードリロード（`Ctrl+F5`）を実施した
4. `GEMDICEBOT_CharaCreator.html` 利用時は、図鑑シート公開設定とURLが有効である

補足:

- キャラ作成HTMLツールは `gviz` 読み込みを優先し、失敗時にCSVへフォールバックします。
- そのため、ネットワーク一時不調や古いキャッシュで一時的に「説明未登録」となる場合があります。
