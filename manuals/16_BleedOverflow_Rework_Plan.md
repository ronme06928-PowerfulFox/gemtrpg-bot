## 1. 目的

`出血氾濫` と `出血遷延` について、方針を次の仕様へ切り替える。

1. **出血氾濫**: 「出血のラウンド終了時処理」を対象に対して即時実行する
2. **出血遷延（Bu-08）**: ラウンド経過で減るバフではなく、**出血ダメージ処理が発生した時に1回消費**される累積デバフにする

本書は、この方針の実装可否・設計・更新対象をまとめる。

---

## 2. 現状調査（実装）

## 2.1 ラウンド終了時の出血処理

`manager/battle/common_manager.py` ではラウンド終了時に以下を実行している。

- 出血値 `bleed_value` だけ HP 減少
- `出血遷延(Bu-08)` が有効なら、出血半減をスキップ
- それ以外は `出血 = floor(出血/2)`

## 2.2 出血氾濫の現状

`plugins/standard.py::BleedOverflowEffect.apply` は現在以下。

- `出血` 値ぶんの `CUSTOM_DAMAGE` を返す
- 出血スタック自体は変更しない
- ラウンド終了時処理（半減 or 維持）とは別ロジック

## 2.3 出血遷延の現状

`plugins/buffs/bleed_maintenance.py` は `lasting` ベースで有効判定している。

- `has_bleed_maintenance`: `delay==0` かつ `lasting>0` で有効
- 消費回数（count）ベースの減衰処理は未実装

---

## 3. 新仕様（確定案）

## 3.1 出血ダメージ処理イベント

「出血ダメージ処理」を次の2経路で発生させる。

1. ラウンド終了時（既存）
2. `出血氾濫` 発動時（新規: 即時）

## 3.2 出血ダメージ処理の共通ルール

出血ダメージ処理1回につき、対象へ次を適用する。

1. `damage = 現在の出血値`
2. HPに `damage` を与える
3. `出血遷延` スタックが1以上なら
   - 出血値は減少しない
   - `出血遷延` スタックを1消費
4. `出血遷延` スタックがなければ
   - 出血値を `floor(出血/2)` にする

## 3.3 出血遷延（Bu-08）の新仕様

- `count`（残回数）を持つ累積デバフとして扱う
- ラウンド経過では減衰しない
- 出血ダメージ処理イベントでのみ `count -= 1`
- `count <= 0` でバフ除去

---

## 4. 実現可能性

結論: **可能（中規模のロジック整理が必要）**

理由:

- 出血処理の実行箇所は既に特定されている（`common_manager.py`）
- `出血氾濫` は既存で Custom Effect 化されており、即時処理への差し替えがしやすい
- `special_buffs` は既に `count` を保持できる設計（`apply_buff`）がある

主な変更点:

- 出血処理を共通関数に抽出
- `Bu-08` の有効判定/消費判定を `lasting` 依存から `count` 依存へ変更

## 5. 推奨実装方針

## 5.1 出血処理を共通化する

新規ヘルパーを追加する（推奨: `manager/bleed_logic.py`）。

推奨API:

```python
def resolve_bleed_tick(char, *, consume_maintenance=True):
    """
    Returns:
      {
        "damage": int,
        "bleed_before": int,
        "bleed_after": int,
        "maintenance_consumed": int,  # 0 or 1
      }
    """
```

## 5.2 ラウンド終了処理を共通ヘルパー経由に置換

`manager/battle/common_manager.py` の既存出血ブロックを置換し、`resolve_bleed_tick()` の結果を反映する。

## 5.3 出血氾濫を「即時出血処理」に置換

`plugins/standard.py::BleedOverflowEffect.apply` を、`resolve_bleed_tick()` の即時実行へ変更する。

返却変更の例:

- `(target, "CUSTOM_DAMAGE", "出血氾濫", damage)`
- 出血変化分がある場合 `(target, "APPLY_STATE", "出血", bleed_after - bleed_before)`
- `出血遷延` 消費ログ

## 5.4 出血遷延（Bu-08）を count 消費型へ変更

`plugins/buffs/bleed_maintenance.py` の変更方針:

- 付与時に `count` を設定（未指定時は1）
- `lasting` ではなく `count` を有効条件にする
- `consume_on_bleed_tick(char)` を追加し、出血処理時に1消費
- `count<=0` で `special_buffs` から除去

実装上の互換対応:

- 旧データ（`lasting` のみ）を読む場合は、`count` 未設定なら `1` とみなす

---

## 6. 仕様上の重要論点

1. **同一ラウンド内で複数回出血処理が起きる**
   - `出血氾濫` 複数回 + ラウンド終了で、`出血遷延` はその都度消費される
2. **出血1の時の扱い**
   - ダメージ1を与え、遷延なしなら出血0へ
3. **出血0の時は何も起きない**
   - ダメージなし、遷延消費なし
4. **ログ粒度**
   - 「出血ダメージ」「出血半減」「遷延消費（残回数）」を分けて出すと検証しやすい
