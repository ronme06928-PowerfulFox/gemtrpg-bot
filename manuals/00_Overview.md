# Gem DiceBot Tool システム定義マニュアル

Gem DiceBot Tool に使用される各種データ（スキル、バフ、アイテム）の定義形式と、システム上の挙動についての総合マニュアルです。

## マニュアル一覧

### 1. [スキル定義と特記処理 (01_Skill_Definitions.md)](./01_Skill_Definitions.md)
`skill_data.json` の構造と、最も複雑な要素である「特記処理 (`Special Effects`)」について解説しています。
- コスト (`cost`)、威力ボーナス (`power_bonus`) の定義
- 状態異常付与 (`APPLY_STATE`)、バフ付与 (`APPLY_BUFF`)
- 特殊効果 (`CUSTOM_EFFECT`) の種類一覧

### 2. [バフ・ステータス定義 (02_Buff_and_Status.md)](./02_Buff_and_Status.md)

- ステータス補正 (`stat_mod`) の定義

### 3. [アイテム定義 (03_Item_Definitions.md)](./03_Item_Definitions.md)
アイテム (`items_cache.json`) の構造について解説しています。
- 回復アイテム (`heal`)

### 4. [輝化スキル定義 (04_Radiance_Skills.md)](./04_Radiance_Skills.md)

## データの反映について

### キャッシュファイル
これらの定義データは、実行時にシステムによって読み込まれ `*_cache.json` というファイルとしてキャッシュされます。
直接キャッシュファイルを編集した場合、**次回起動時に上書きされる可能性**があります（元データがスプレッドシート等にある場合）。
開発環境やローカル運用では、キャッシュファイルを直接編集してテストすることが可能です。

### 特記処理の記述
スキルの `特記処理` 欄は、JSON形式の文字列である必要があります。記述ミス（カンマ忘れ、括弧の不整合など）があると、スキル読み込み時にエラーとなるか、効果が発動しません。
作成時は JSON Validator などで構文チェックを行うことを推奨します。
